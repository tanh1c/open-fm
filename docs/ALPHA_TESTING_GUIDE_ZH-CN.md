# Openfoot Manager — Alpha 测试指南

你好，欢迎来到 **Openfoot Manager** 的 Alpha 测试！首先，真的非常感谢你愿意参与。愿意试玩一个尚未完成的游戏，并帮助我们把它做得更好，这对我们来说意义重大。

这份指南会介绍这款游戏是什么、目前哪些内容已经可玩、哪些内容还没有完成，以及你可以怎样最有效地帮助我们。

## Openfoot Manager 是什么？

Openfoot Manager 是一款**开源足球经理模拟游戏**。你可以把它看作是写给经典足球经理游戏类型的一封情书: 你将接手一家俱乐部，管理阵容，设定战术，处理转会，并带领球队完成一个完整的联赛赛季。

它以桌面应用的形式构建，使用 [Tauri](https://tauri.app/) 技术栈开发(Rust 后端 + React 前端)，因此可以在 Windows、macOS 和 Linux 上原生运行，不需要浏览器，也不需要联网。

### 这次 Alpha 里已经有的内容

你现在可以体验这些内容:

- **创建一名主教练**，填写姓名、出生日期和国籍
- **选择一支球队**，来自一个自动生成的 16 支球队联赛
- **管理你的阵容**，包括设置阵型、选择首发 11 人、指定定位球主罚者
- **训练球员**，调整训练重点、强度和每周计划
- **招聘和解雇工作人员**，包括教练、理疗师、球探和助理教练
- **进行比赛**，可以实时逐分钟指挥，也可以交给助理教练处理
- **中场训话**和**赛后新闻发布会**，会影响球队士气
- **阅读新闻**，包括比赛报道、联赛综述和积分榜更新
- **处理收件箱**中的消息，查看来自教练组、董事会和各种游戏事件的信息
- **考察球员**并**进行转会操作**
- **查看财务**，包括工资、预算和收入
- **完成一个完整赛季**并进入下一赛季

### 这次 Alpha 里还没有的内容

为了设定正确预期，以下内容已经规划，但暂时还没有实装:

- 多级联赛 / 升降级
- 杯赛
- 球员合同谈判
- 青训营
- 更细致的比赛战术(盯人、防守定位球套路等)
- 多人模式
- 音效 / 音乐
- 第一周之后更完整的新手教程 / 引导

这些内容以后都会逐步加入。但现在，我们更需要你帮助我们找到现有内容里的 bug、设计粗糙点和不顺手的地方。

## 快速开始

### 安装

1. 通过我们提供给你的链接下载对应平台的安装包
2. 运行安装程序。如果你在 Windows 上看到 SmartScreen 警告，这是因为应用目前还没有签名。点击“更多信息”→“仍要运行”
3. 启动 **Openfoot Manager**

### 你的第一局游戏

1. 在主菜单点击 **New Game**
2. 填写主教练信息(姓名、出生日期、国籍)
3. 选择一个世界数据库(默认的 “Random World” 就可以)
4. 从列表中挑选一支球队，看看它们的属性，选一支你觉得有意思的
5. 进入 **Dashboard**，这里就是你的主控页面

### 需要知道的几个重点

- 右上角的 **Continue** 按钮会让时间前进一天。旁边的下拉箭头可以让你直接跳到下一个比赛日。
- 左侧的**侧边栏**是主要导航区域，你可以在这里进入阵容、战术、训练、赛程、财务等页面。
- 比赛开始前，你会被问到想怎么进行比赛: **Go to the Field**(完全控制)、**Watch as Spectator**(观看 AI 比赛) 或 **Delegate to Assistant**(直接交给助理教练处理结果)。
- 在比赛过程中，你可以换人、调整阵型、修改比赛风格，并重新指定定位球主罚者。
- 比赛结束后会有中场 / 全场训话，以及可选的赛后发布会。
- 当你退出到主菜单时，游戏会**自动保存**。你也可以在设置中手动保存。

## 我们最需要你提供什么

### 简短版本

玩游戏。试着把它玩坏。然后告诉我们发生了什么。

### 详细版本

我们主要想收集三类反馈:

#### 1. Bug 和崩溃

这是最高优先级。如果某个功能坏了、卡死了、崩溃了，或者行为明显不对，我们都想知道。

例如:

- 我一尝试开始比赛，游戏就崩溃了
- 我修改的阵型在重新进入游戏后没有保存
- 某个球员显示受伤，但仍然出现在首发阵容里
- 第 5 个比赛日之后积分榜计算不正确
- 我收到了一条关于尚未发生比赛的消息

#### 2. 易用性问题

有些东西没坏，但它们让人困惑、烦躁或者难以上手。

例如:

- 我找不到在哪里修改首发阵容
- 训练页面信息太多了，我不知道每个选项具体有什么作用
- 文字在我的屏幕上太小 / 太大
- 我不明白“比赛风格”到底会影响什么
- 收件箱里都是我不关心的消息

#### 3. 平衡性和整体体验

这部分更主观，但同样重要。游戏整体**感觉**对不对？

例如:

- 我的球队每场都 5 比 0 赢球，太简单了
- 训练似乎完全没有效果
- 无论我做什么，球员士气都没有变化
- 转会报价总是被拒绝
- AI 球队显得太弱 / 太强

## 如何提交反馈

最简单的方式是使用我们的 **GitHub Issue 模板**。选择合适的模板并填写即可，**你可以直接用中文提交**。

如果你想和开发团队或其他玩家交流，也可以加入 Discord 服务器：https://discord.gg/4ppEDH68

- [**Bug 报告**](https://github.com/openfootmanager/openfootmanager/issues/new?template=bug_report_zh_cn.yml) — 某些内容崩溃了、出错了，或行为明显不正确
- [**反馈 / 建议**](https://github.com/openfootmanager/openfootmanager/issues/new?template=feedback_zh_cn.yml) — 易用性问题、平衡性意见或新的想法
- [**游戏会话报告**](https://github.com/openfootmanager/openfootmanager/issues/new?template=session_report_zh_cn.yml) — 总结你本次游玩的整体体验(非常有价值)

### 日志文件

如果你在提交 bug，**请尽量附上日志文件**。日志会详细记录问题发生时游戏正在做什么，很多时候它们能让我们在 5 分钟内修复问题，而不是花几个小时去复现。

**日志位置如下:**

- **Windows:** `C:\Users\<你的用户名>\AppData\Roaming\com.sturdyrobot.openfootmanager\logs\`
- **macOS:** `~/Library/Application Support/com.sturdyrobot.openfootmanager/logs/`
- **Linux:** `~/.local/share/com.sturdyrobot.openfootmanager/logs/`

请把整个 `logs` 文件夹打包成 zip 后附加到反馈中。日志不包含个人隐私信息，只会记录游戏事件、命令和错误堆栈。

## 已知问题

以下问题我们已经知道了，你不需要重复提交(当然，如果它们严重影响了你的体验，欢迎补充说明):

- **目前没有音效或音乐**，游戏暂时是静音的
- **界面缩放**在部分屏幕尺寸上可能还不够理想。如果显示不合适，请前往 Settings → Display → UI Scale 调整
- **部分文本还没有翻译完成**，国际化仍在持续推进中
- **当前 Alpha 的存档未来可能不兼容**，所以先不要太依赖这些存档
- **连续模拟很多天时性能可能会略微下降**
